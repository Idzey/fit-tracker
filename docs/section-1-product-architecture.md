## 1.2 Разбивка по фичам

### Trainer features

| Фича | Описание |
|------|----------|
| **Client management** | Добавление клиентов по email-инвайту или вручную; просмотр профиля, статуса, истории |
| **Workout builder** | Конструктор шаблонов: дни, упражнения (название, подходы, повторения, вес, описание) |
| **Program assignment** | Назначение шаблона клиенту с датой начала |
| **Progress tracking** | Просмотр выполненных тренировок и прогресса клиентов |
| **Photo gallery** | Просмотр фото прогресса клиентов по датам |
| **Realtime dashboard** | Live-уведомление когда клиент завершил тренировку |
| **Subscription management** | Просмотр плана, переход на Pro, история платежей |

### Client features

| Фича | Описание |
|------|----------|
| **Workout feed** | Список назначенных тренировок на сегодня / на неделю |
| **Workout execution** | Пошаговое выполнение: отметка подходов, ввод фактического веса |
| **Progress history** | Графики прогресса по упражнениям, история тренировок |
| **Photo upload** | Загрузка фото с датой для отслеживания изменений тела |
| **Push notifications** | Напоминания о тренировках по расписанию |
| **Profile** | Личные данные, цели, параметры тела |

## 1.4 Стратегия монетизации

### In-App Purchase vs Web-платёж

| Параметр | RevenueCat (IAP) | ЮMoney (web) |
|---|---|---|
| Комиссия | 30% App Store / 15–30% Google Play | ~3.5% |
| Правила сторов | Обязателен для цифровых сервисов внутри приложения | Запрещён для цифровых покупок через in-app UI |
| Реализация | SDK + нативный paywall | Redirect на web-страницу оплаты |
| UX | Нативный (Touch ID / Face ID) | Выход из приложения |
| Тестовая среда | Sandbox (TestFlight / Google Play Test) | Тестовый режим ЮMoney |
| Поддержка РФ | Ограничена (App Store RU приостановлен в 2022) | Нативная |

### Принятое решение для проекта

**Гибридная схема:**

1. **RevenueCat** — для iOS и Android IAP. Это требование сторов: если пользователь может купить подписку внутри приложения, магазин обязывает использовать нативный IAP. RevenueCat абстрагирует различия между App Store и Google Play, предоставляет единый webhook и SDK.

2. **ЮMoney** — для web-токена (оплата через браузер по прямой ссылке в email). Позволяет обойти комиссию сторов для пользователей, которые платят напрямую. Актуально для российского рынка.

```
Тренер нажимает "Upgrade" в приложении
       │
       ├─► iOS/Android ──► RevenueCat paywall ──► App Store/Google Play IAP
       │                         │
       │                   webhook → backend → обновить план
       │
       └─► email-ссылка ──► web-страница ──► ЮMoney ──► webhook → backend
```

> **Упрощено для портфолио:** webhook-обработка реализована, но полная защита от replay-атак и идемпотентность — задача продакшн-уровня. В демо используются sandbox-события.

## 1.5 User Journeys

### Journey 1: Onboarding тренера

```
Регистрация (email + пароль + роль TRAINER)
  └─► Заполнение профиля (имя, фото, специализация)
        └─► Dashboard (пустой, CTA "Добавить клиента")
              └─► Добавление клиента → отправка инвайта
                    └─► Создание первого workout template
                          └─► Назначение программы клиенту
```

### Journey 2: Onboarding клиента

```
Получение инвайта от тренера (ссылка / код)
  └─► Регистрация с привязкой к тренеру
        └─► Заполнение профиля (цели, параметры)
              └─► Home (список назначенных тренировок)
                    └─► Первая тренировка
```

### Journey 3: Выполнение тренировки (клиент)

```
Home → Today's workout
  └─► Открыть тренировку (список упражнений)
        └─► Упражнение 1: ввод веса/повторений → отметить подход ✓
              └─► Все подходы выполнены → следующее упражнение
                    └─► Все упражнения выполнены → "Завершить тренировку"
                          └─► Тренер получает realtime-уведомление
                                └─► Progress экран обновляется
```

### Journey 4: Upgrade подписки (тренер)

```
Попытка добавить 4-го клиента (Free plan)
  └─► Paywall экран (план, цена, фичи)
        ├─► iOS/Android: нативный IAP (RevenueCat)
        └─► Web: ЮMoney (redirect)
              └─► Успешная оплата → webhook → план обновлён
                    └─► Клиент добавлен
```

### Journey 5: Загрузка фото прогресса (клиент)

```
Progress → Photos → "+" 
  └─► Выбор фото из галереи / камеры
        └─► Preview + подтверждение
              └─► Upload: presigned URL → R2
                    └─► Фото появляется в галерее клиента и тренера
```

## 1.6 Границы MVP

### Входит в MVP (Sprints 0–5)

- Регистрация/логин обеих ролей (email + пароль)
- Управление клиентами (CRUD, инвайт по email)
- Создание и назначение workout templates
- Выполнение тренировок клиентом (workout execution flow)
- Загрузка фото прогресса (presigned URLs, R2)
- Push-уведомления (напоминания + completion alert)
- Realtime dashboard для тренера (SSE)

### Post-MVP (Sprints 6–7)

- Подписки и биллинг (RevenueCat + ЮMoney)
- UX polish (skeleton screens, haptics, анимации)
- Offline-first (action queue, persist cache)

### Roadmap (не реализуется)

| Фича | Причина отложить |
|------|-----------------|
| Web-интерфейс для тренера | Отдельный фронтенд-проект |
| Видео-упражнения | Требует CDN, video player, большое хранилище |
| Аналитика / дашборды | Нет в требованиях |
| AI-рекомендации | Out of scope для Mid-портфолио |
| Multi-trainer per client | Усложняет модель данных и permissions |
| i18n | Не критично для портфолио |
